Efficient and Privacy-Preserving Encode-Based
Range Query Over Encrypted Cloud Data
Yanrong Liang , Jianfeng Ma , Member, IEEE, Yinbin Miao , Member, IEEE,
Yuan Su , and Robert H. Deng , Fellow, IEEE

Abstract— Privacy-preserving range query, which allows the
server to implement secure and efficient range query on
encrypted data, has been widely studied in recent years. Existing
privacy-preserving range query schemes can realize effective
range query, but usually suffer from the low efficiency and
security. In order to solve the above issues, we propose an
Efficient and Privacy-preserving encode-based Range Query over
encrypted cloud data (namely basic EPRQ), which encodes
the data and range by using Range Encode (REncoder), and
then encrypts the codes via Additional Symmetric-Key Hidden
Vector Encryption (ASHVE) technology. The basic EPRQ can
achieve effective range query while ensuring privacy protection.
Then, we split the codes to reduce the storage cost. We further propose an improved scheme, EPRQ+, which constructs a
binary tree-based index to achieve faster-than-linear retrieval.
Finally, our formal security analysis proves that our schemes
are secure against Indistinguishability under Chosen-Plaintext
Attack (IND-CPA), and extensive experiments demonstrate that
our schemes are feasible in practice, where EPRQ+ scheme
improves the storage efficiency by about 4 times and the query
efficiency by about 8 times compared to the basic EPRQ.
Index Terms— Searchable symmetric encryption, privacypreserving range query, range encoder, symmetric-key hidden
vector encryption.

to the length of code. When the total query range is very large,
prefix code will suffer from low query efficiency. To improve
the query efficiency, the researchers map all elements in prefix
family to a bloom filter [7], [9]. However, the bloom filter
will suffer from false positives, resulting in incorrect query
results. The B+ tree [11] needs to construct a tree structure
for all data as shown in Fig. 1. The values of each tree node
represent a number range, where all values in the left subtree
are smaller than or equal to the value of this tree node, and
all values in the right subtree are larger than the value of this
tree node. The user realizes the query by judging the size of
query range and value of each tree node [12]. Although B+
tree can achieve efficient range query, it will reveal the size
relationship of each node value. Therefore, the existing range
query method has the problem of inefficiency or inaccuracy or
revealing some privacy. An accurate and efficient range query
method is urgently needed.

To protect data privacy, the EPRQ scheme needs to adopt
some encryption technology to ensure the security of scheme.
At present, the mainstream encryption mechanism used in
EPRQ scheme is k-Nearest Neighbor (kNN) [13], traditional
asymmetric encryption and Homomorphic Encryption (HE)
technologies. To enhance the commonality of query scheme
and facilitate encryption, researchers usually quantize the
value and the query range into two vectors, and use the
vector product to judge the relationship between the value
and the range. kNN is utilized to implement EPRQ [14],
[15], which encrypts vectors by random matrix multiplication.
However, kNN has been proven not to be resistant to Know
Plaintext Attacks (KPA) [16], [17]. In addition, researchers
also use traditional asymmetric encryption technology based
on difficulty problems to implement EPRQ [18]. However,
these schemes need to invoke bilinear map technique, which
will result in larger search costs. In addition, the schemes [19],
[20] use HE to realize query by directly judging the size
of the value and range endpoint. However, HE will result
in greatly storage and compute costs, which is unacceptable
to mobile users. Therefore, the existing privacy-preserving
range query scheme has the problem of insufficient security or
low query efficiency. How to find a privacy-preserving range
query scheme that balances security and efficiency is what
researchers need to consider.
As mentioned above, the existing range query schemes
still suffer from the low efficiency and security. To solve
the above issues, we first dexterously combine the idea
of REncoder and Additional Symmetric-key Hidden Vector
Encryption (ASHVE), and propose an Efficient and Privacypreserving Range Query scheme (basic EPRQ). Specifically,
the basic EPRQ uses REncoder to convert each data and
query range into a fixed length binary string, and improves
SHVE to encrypt the valid bits to achieve secure range query.
Since REncoder conversion will generate a large number of
redundant meaningless values, we split binary strings to reduce
the storage cost. At the same time, to improve the query
efficiency, we reduce query complexity by constructing the
tree structure for all data. On this basis, we further design an
enhanced scheme EPRQ+, which supports lower cost range
query. The main contributions of our work are as follows:
• We use REncoder to convert the values and query range
into two fixed length binary strings, and encrypt the
converted binary strings using ASHVE to construct an
new encrypted index structure.

II. R ELATED W ORK
A. Searchable Encryption (SE)
SE can achieve efficient retrieve of encrypted data without
compromising the privacy information of encrypted data and
query [21], [22]. Song et al. [23] first proposed the concept of
Searchable Symmetric Encryption (SSE), and also proposed
the first SSE scheme based on ciphertext full-text search.
Boneh et al. [24] first proposed the concept of asymmetric
searchable encryption (Public-Key Encryption with Keyword
Search, PEKS), and constructed the first PEKS scheme by
using bilinear map. On this basis, to satisfy various demands
of users, SE schemes that can realize various functions have
been gradually proposed. Golle et al. [25] proposed a PEKS
scheme for the first time that can realize conjunction keyword search, which greatly improves the accuracy of search
results. To improve the retrieval efficiency, Curtmola et al. [26]
improved the index structure of SE and designed the first SE
scheme based on inverted index. Tong et al. [27] used Bloom
filter and Katz-Sahai-Waters (KSW) encryption to implement privacy-protecting Boolean range query, and adopted
Ciphertext-Policy Attribute-Based Encryption (CP-ABE) to
achieve fine-grained temporary access control to further ensure
the security of encrypted data. Liang et al. [28] used an
improved Bloom filter structure CSC to achieve efficient
retrieval, which overcomes the unavoidable high false positives and identifier correlation in traditional Bloom filters.
To prevent malicious cloud servers from forging some search
results, Li et al. [29] adopted homomorphic MAC and random challenge technologies to verify the correctness and
completeness of the returned results respectively. Most of
the above schemes focused on text-keyword retrieval. With
the development of science and technology, various kinds
of data have emerged, such as image data, spatial data

B. Privacy-Preserving Range Query
Given an encrypted query range and a set of encrypted
values, privacy-preserving range query can return all encrypted
values that contained within this encrypted query range without revealing the query range and values. At present, range
query has been widely studied because of its wide applicability. Tong et al. [14] and Song et al. [33] encoded the
query range and value into two vectors using comparable
product encoding and curve fitting, respectively, and then
both used the improved kNN to encrypt vectors and achieve
retrieval. In addition, the scheme [14] used CP-ABE to
encrypt the secret key used by each outsourced ciphertext
to protect the ciphertext privacy, and the scheme [33] used
polynomial function to set the access policy to protect the
ciphertext privacy. Gong et al. [15] converted spatial data
into vectors by Gray code and Bloom filter, and used the
improved kNN to encrypt vectors and implement boolean
range query. All of the above schemes can realize effective
range query. However, since kNN has been proved to not
resistant to Known Plaintext Attacks (KPA) [16], [17], the
above schemes suffered from inadequate security. To improve
the security of the query scheme, Guan et al. [18] implemented rectangle range intersection detection based on an
inner-product preserving encryption, and used bilinear map
and Bloom filter to implement encryption. Wang et al. [19]
directly used symmetric homomorphic encryption (SHE) to
encrypt the query range and values, and realized range query
by judging the size of ciphertexts with the help of two
servers. Tong et al. [27] and Miao et al. [20] converted
spatial data into vector through the specific transformation
methods, and realized secure retrieval by using KSW and
SHE respectively. Although the above schemes can achieve
secure range query, SHE or traditional symmetric encryption
technology cannot avoid a lot of computation costs, which
will result in inefficient query. To achieve a secure and
efficient range query, Wang et al. [7] and Miao et al. [34]
used Symmetric-key Hidden Vector Encryption [35] to encrypt
vectors converted from data and query range. However, both
schemes used Bloom filter to improve query efficiency, which
will introduce false positive that Bloom filter cannot avoid
and produce incorrect query results. In addition, to improve

the practicality of scheme, some researchers consider implementing range query on dynamic database [8], [36]. To ensure
the forward/backward privacy of dynamic range query, two
schemes [8], [36] embeded the number of updates and queries
into indexes and tokens, and assigned state to each entity in
the system. Only when the numbers and status match at the
same time, the server can successfully query, so as to realize
the forward/backward privacy of dynamic range query.
A comparative summary of our EPRQ schemes and the
existing schemes is presented in TABLE I. It can be
seen that, although the existing schemes can realize the
privacy-preserving range query, they have their own shortcomings. Wang et al. [7], Tong et al. [14], Guan et al. [18],
and Tong et al. [27] can achieve sublinear query efficiency.
However, Wang et al. [7] and Guan et al. [18] have the false
positive problem that Bloom filter has to face. The k-Nearest
Neighbor encryption algorithm and R-tree structure used by
Tong et al. [14] are not secure enough and will leak privacy
information. The traditional symmetric encryptions adopted by
Guan et al. [18] and Tong et al. [27] involve computationally expensive operations and result in low query efficiency.
In addition, although Song et al. [33] and Miao et al. [34] can
realize efficient range query, the kNN encryption algorithm
used by Song et al. [33] has the problem of insufficient security, and Miao et al. [34] has the false positive problem faced
by Bloom filter, and cannot realize accurate and secure range
query. Therefore, the target of this paper is to construct the
privacy-protecting and efficient accurate range query scheme.

IV. P ROBLEM F ORMULATION
In this section, we formalize the system model & threat
model, problem definition and design goals of our schemes.
A. System Model & Threat Model
In this paper, we consider a scenario where the encrypted
data is outsourced to the cloud server. As shown in Fig. 3,
our scheme involves three entities, namely Data Owner (DO),
Data User (DU), and Cloud Server Provider (CSP). The roles
of each entity are as follows:
• Data owner. DO is responsible for constructing and
encrypting indexes of all outsourced data, and sends it
to CSP. At the same time, DO generates the secret key
for all authorized DUs and distributes them by a secure
channel.
• Data users. The authorized DU can access the outsourced
encrypted data. When an authorized DU needs to query,
he or she encrypts the query range as a token and submits
the token to the cloud server.

B. Problem Definition
DB = {d1 , · · · , dn } represents the dataset outsourced by
DO and q = [qa , qb ] represents the query range transmitted
by DU. To achieve query, the data and query range are
usually encoded into two codewords F and S using a specific
encode method, and then by matching F and S to determine
whether the data is included in the query range. Existing
range query schemes usually use prefix encode to implement
query. As shown in Fig. 1, an m-bit data di = (o1 o2 · · · om )
needs to construct a prefix family containing m+1 elements
F = {(o1 o2 · · · om ), (o1 o2 · · · ∗), · · · , (o1 ∗ · · · ∗), (∗ ∗ · · · ∗)}.
When q is queried, it is converted to the smallest prefix
element S that can contain the query range. The cloud server
needs to match elements in the prefix family and prefix
element one by one, that is, whether S is contained in F
and m+1 matches are required. The computational complexity
of the query is proportional to the size of m, which makes
the query time too large under large datasets. In addition,
although the converted codeword is not the original data and
range, it will still reveal relevant privacy. Therefore, EPRQ
needs to encrypt codeword to achieve secure query. In the
existing EPRQ schemes, the kNN algorithm is usually used
for encryption. However, kNN algorithm has been proven not
to be resistant to Know Plaintext Attacks (KPA), which cannot
achieve sufficient security. Next, we give the formal definition
of our scheme as follows:

C. Design Goals
To achieve efficient and privacy-preserving range query, our
scheme needs to realize the following requirements:
• Data Privacy: DO constructs the index for the dataset
and outsources them to the cloud server. In addition, the
unauthorized entities and cloud server cannot know the
specific content of the plaintext dataset.
• Query Privacy: Only the authorized DU can generate
token and initiate query. In addition, the unauthorized
entities and cloud server cannot know the query specific
content.
• Efficiency: Our scheme can achieve efficient and
privacy-preserving range query through indexes and
tokens.
V. O UR P ROPOSED S CHEME
In this section, we propose two range query schemes: Efficient and Privacy-preserving encode-based Range Query over
encrypted cloud data (basic EPRQ) and the enhanced scheme
EPRQ+. Before introducing the specific EPRQ, we first
improve the traditional SHVE to propose ASHVE by introducing additional values, which can hide the index length. After
that, we propose the basic EPRQ by combining REncoder with
ASHVE. Then, to reduce storage requirements and improve
retrieval efficiency, we propose a more practical enhanced
scheme EPRQ+ by using split and tree structure.

A. Basic Scheme: Basic EPRQ
Main Idea. As mentioned in Section I, the existing EPRQ
schemes suffer from some challenges, such as inefficiency
and insecurity. The existing EPRQ schemes usually use prefix
encode to implement query. However, the prefix encode needs
to match prefix family F of d = (o1 o2 · · · om ) and prefix
element S of q = [qa , qb ] in query, where m + 1 matches
need to be performed and m is linearly related to the total
query range. When the total query range is large, prefix encode
suffers from low query efficiency. Therefore, we need to
construct a novel index structure to achieve efficient range
query. We take advantage of a new binary tree-based encode
method: REncoder. We frist convert the m-bit data or query
range through REncoder as a 2(m+1) − 1-bit binary string.
Then, the cloud server only needs to match two binary strings
to judge whether the data is included in the query range,
which can achieve more efficient range query than prefix
encode. However, unencrypted binary strings will reveal the
value of the original data. To protect the security of binary
strings, we can directly call the traditional SHVE algorithm
for encryption. Nevertheless, the traditional SHVE will reveal
the length of the binary string. According to the binary tree
conversion method and the encryption method of SHVE,
when one query is successful matched, the adversary will
infer the specific query range, which still reveals the privacy.
Therefore, to solve the privacy leakage problem mentioned
above, we can hide the specific binary string length and
improve the traditional SHVE by introducing some additional
positions to present Additional Symmetric-key Hidden Vector
Encryption (ASHVE). We use ASHVE to encrypt the above
converted binary strings and achieve the secure range query.

Remark: As shown in subsection V-A, our basic EPRQ
can achieve efficient range query on encrypted data, and also
achieve the privacy protection by encrypting data and query
range. However, the basic EPRQ still has a shortcoming in
terms of storage. The m-bit binary is converted to a (2(m+1) +

increases exponentially with the length of the binary. When
m is large, the binary string will also be long. At the same
time, each binary string has a large number of 0 value without
meaning. The large storage requirements are a challenge for
both user and server. In addition, the basic EPRQ needs to
query the encrypted data stored in the cloud server one by
one. Despite the efficiency of single query is high, when a
large amount of data is stored in the cloud server, DU will
still face a lot of query latency, which will greatly affect the
user’s query experience.

VI. S ECURITY A NALYSIS
In this section, we first give the formal security definition
to prove that EPRQ is secure against IND-CPA [34]. Then,
we give two theorems to prove the security of our EPRQ
schemes.
Before proving that our EPRQ schemes guarantee data
privacy and query privacy against IND-CPA, we first simulate
a security game played between an adversary A from the ideal
security and a challenger C.
Definition 2 (IND-CPA of EPRQ): Let 5 = (Setup,
IndexBuid, TokenGen, Query) be an EPRQ scheme. The
security game between an adversary A and a challenger C is
defined as:
• Init: Input a security parameter λ, A selects two
datasets DB0 = {d01 , d02 , · · · , d0n } and DB1 =
{d11 , d12 , · · · , d1n } or two query range q0 = [q0a , q0b ]
and q1 = [q1a , q1b ] to C, where each dataset has n data.
• Setup: The challenger C runs Setup to generate the
master key msk and keeps msk private.
• Phase 1: The adversary A issues a series of query
requests, which can be divided into the following two
parts:
1) On the j-th index request, the adversary A outputs a
dataset DB j = {d j1 , d j2 , · · · , d jn } and the challenger C
runs IndexBuid to generate an encrypted index EDB j in
response.
2) On the j-th token request, the adversary A outputs a
query range q j = [q ja , q jb ] and the challenger C runs
TokenGen to generate a token TQj in response.

A. Theoretical Analysis
We analyze the computation and storage costs of

the multiplication of encrypted matrixes and judges the size
relationship between the product and 0 to deduce whether the
object is included in the query range.

VIII. C ONCLUSION
In this paper, we propose two efficient and privacypreserving range query schemes. Firstly, we utilize REncoder
to transform the data and implement retrieve simultaneously.
Then, we use the improved ASHVE to encrypt the converted
binary and propose the basic EPRQ scheme. In addition,
an improved scheme EPRQ+ is proposed through split operation and tree structure, which further reduces the storage
requirement and speeds up the query speed of the basic
EPRQ. We have rigorously analyzed the security of our EPRQ
schemes under IND-CPA. Finally, the performance analysis
shows that our EPRQ schemes can realize efficient range
query. In the future, we would like to construct dynamic range
query scheme that can guarantee forward/backward privacy.
